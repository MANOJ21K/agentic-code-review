# Contributing to agentic-code-review

Thank you for your interest in contributing to `agentic-code-review`! We want to make contributing to this project as easy and transparent as possible.

## How to Contribute

Whether it's a bug report, new feature, correction, or additional documentation, we greatly value feedback and contributions.

### 1. Find an Issue
We encourage you to look through the GitHub issues. If you are new to the project, look for issues labeled:
- `good first issue`
- `help wanted`

These are specifically curated to be accessible and provide a great entry point into the codebase.

### 2. High-Impact Areas for Contribution
If you're looking for ideas on where to contribute, here are a few areas we are actively looking to improve:
- **Adding support for new languages:** Extending reviewer capabilities to understand nuances of additional programming languages.
- **Optimizing agent prompts:** Refining the system prompts for specific reviewers (e.g., Security, Bugs) to reduce false positives and improve confidence scoring.
- **Improving orchestration:** Enhancing how context is passed between the orchestrator and subagents.

### 3. Local Development Setup
1. Fork the repository and clone it locally.
2. Make sure you have Claude Code installed.
3. Test your changes locally by linking your modified plugin.

### 4. Pull Request Process
- Ensure any install or build dependencies are removed before the end of the layer when doing a build.
- Update the README.md with details of changes to the interface, this includes new environment variables, exposed ports, useful file locations and container parameters.
- Provide a clear description of the problem you are solving and how your PR addresses it.

## Code of Conduct
Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms. Let's build a welcoming and inclusive community.
